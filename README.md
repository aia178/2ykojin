# SaveWish — IoT貯金箱アプリ

## 概要

SaveWish は、IoT貯金箱と連携するAndroid向けの貯金管理アプリです。  
「欲しいものを目標に設定して、楽しく貯金する」をコンセプトに開発しました。

## 動作フロー
1. 貯金箱に硬貨を入れる
2. ロードセルが重さを測定し、ESP32が硬貨の種類と金額を判定する
3. ESP32がHTTP通信によりFirebaseに入金データを送信する
4. アプリがFirebaseからリアルタイムでデータを取得し、貯金額や進捗を更新する

## なぜこのアプリを作ったのか

自分自身に浪費癖があり、それを少しでも解消できる方法はないかと考えたのがきっかけです。  
ただ節約しようと思うだけではモチベーションが続かないため、欲しいものを具体的な目標として設定し、貯金の進捗が目に見える形で分かるアプリがあれば、楽しみながら貯金を続けられるのではないかと考えました。  
さらに、IoT貯金箱と連動させることで、物理的にお金を入れるたびにアプリ上で進捗が更新される仕組みを実現し、貯金のモチベーションを維持できるようにしました。

## 主な機能
アプリ内で楽天市場の商品を検索して、そのまま貯金目標として設定できます。
自分で目標名・金額・画像を自由に設定するカスタム目標にも対応しています。
ホーム画面では現在の貯金額・目標金額・残額・達成率がリアルタイムで確認できます。
複数の目標を同時に管理でき、ワンタップで表示する目標を切り替えられます。
貯金履歴を一覧で確認でき、累計貯金額の推移を折れ線グラフで見ることもできます。
目標金額に達成すると、お祝いのダイアログが表示されます。
今月の貯金額や総貯金額などの統計情報も確認できます。

## AIを使用した箇所と使い方

自分はデザインが苦手なため、主にアプリ側のデザイン面で使用しました。
コードを書くのはできるだけ自分で行い、AIにはコードの書き方を相談したり、コードの修正を提案してもらったりしました。
またデバッグやエラー解決の際にもAIを使用しました。


### AIの使い方のポイント

1. 設計は自分で行い、実装をAIに支援してもらう方針で開発しました。アプリの全体構成・機能要件・画面遷移は自分で設計し、各画面の実装や技術的な課題の解決にAIを活用しました。
2. コードの大枠を自分で書き、詳細な処理部分をAIに穴埋め式で補完してもらうことで、コードの全体像を把握しながら効率的に実装しました。
3. AIが生成したコードをそのまま使うのではなく、内容を理解した上で修正・調整しながら取り入れました。
4. 対話的にAIを使用し、1回の回答で完結させるのではなく、段階的に質問・修正を繰り返して完成度を高めていきました。

## 苦労した点・実装できなかった点
硬貨の重さを正確に測定することが難しく、何度もサンプルを測定して調整を行いました。
しかし、すべての硬貨で安定動作させることが難しかったため最終的には1円・100円・500円玉のみ対応することにしました。
今後の発展としてカメラでの認識にすることによって、精度は難しくなると思いますが、より多くの硬貨や紙幣に対応できるようになると考えています。
また、本当はamazonのAPIを使いたかったのですが、条件があり楽天APIを使用しました。
